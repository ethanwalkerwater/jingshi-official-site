# 菁仕官网重设计方案 v1

> 状态：待确认。确认后按本文档开发。
> 目标：去 AI 模板感、统一配色、移动端阅读效率优先。

---

## 1. 设计方向：暖纸质感 · 编辑排版（Warm Paper Editorial）

**一句话**：像一份排版精良的院校手册，而不是一张深色科技风落地页。

摆脱 AI 味的三条铁律：

1. **浅色为底，深色为字**。整页以暖白/米白为基底，藏青只作为文字与关键 UI 颜色出现，不再大面积铺深色背景（现状全页 >50% 是深海军蓝，是"AI 味"的最大来源）。
2. **一个强调色，克制使用**。金色保留（与 logo 一致），但只做平面纯色、只用于：主 CTA 按钮、评分数字、个别装饰细线。禁止金色渐变、发光阴影、金边框 hover。
3. **信息密度优先于留白仪式**。缩小板块间距、去掉每板块重复的"eyebrow 小字 + 大标题 + 大段副文案"三件套，副文案能删就删，删不掉就压缩到一行。

## 2. 色彩系统

| 角色 | 色值 | 用途 |
|---|---|---|
| 底色 paper | `#FBFAF7` | 全站主背景 |
| 底色 alt | `#F3F1EA` | 相邻板块交替，仅轻微区分 |
| 墨色 ink | `#182741` | 标题、正文主色（取自 logo 藏青） |
| 墨弱 ink-soft | `#4A5568` | 次要文字 |
| 金 gold | `#B08D46` | 仅：主 CTA、评分、细装饰。平面纯色 |
| 卡片 card | `#FFFFFF` | 卡片底，1px 浅边，无大投影 |
| 线 line | `rgba(24,39,65,.10)` | 分隔线、卡片边 |

- 比例约束：浅底色 ≥ 80%，墨色文字/UI ≈ 15%，金色 ≤ 3%。
- 阴影一律用极浅扩散阴影或直接不用；禁止彩色光晕。
- 圆角统一两档：卡片 12px，按钮/胶囊 999px。

## 3. 字体与排版

- **弃用 Google Fonts**（微信内加载不稳定、阻塞首屏），改系统字体栈：
  `PingFang SC / Hiragino Sans GB / Microsoft YaHei / system-ui`。
  英文与数字用系统栈即可，不再引入 Space Grotesk。
- 标题全部无衬线，靠字重（700/800）与字号对比建立层级（现状 `--font-serif` 名存实亡，一并清理）。
- 字级（移动端 / 桌面端）：
  - H1：30 / 44px，行高 1.25
  - H2 板块标题：24 / 32px
  - 正文：15–16px，行高 1.75
  - 辅助小字：12–13px
- 图标：弃用 jsDelivr 整包 tabler webfont，改内联 SVG（全站实际只用约 6 个图标：星、箭头、菜单、关闭、电话、日历）。

## 4. 间距与节奏

- 4 的倍数体系。板块上下间距：移动端 48px / 桌面 80px（现最高 124px，过大）。
- 首屏 Hero 高度：移动端自适应内容（约 70vh 上限）/ 桌面 560–640px，不再 92vh。
- `.wrap` 最大宽 1120px，移动端左右边距 20px。

## 5. 首页新结构（板块顺序调整）

现状：Hero → 名师轮播 → 理念 → 课程 → CTA（深-浅-深-深 色块拼接）
改为：

1. **Hero（浅色）**：左文右图（移动端上文下图）。标题 + 一句副文案 + 单个主 CTA「预约免费咨询」+ 三个关键数据（20+ 师资 / 5 校区 / 一站式）以小字一行呈现，不再做独立统计条。背景图只在此处出现一次，全站不再复用同一张图。
2. **名师团队**：静态网格，不做自动轮播。
   - 桌面 4 列，平板 3 列，**手机 2 列**；
   - 首页展示 8 位 featured 老师 + 「查看全部 20 位老师」文字链。
3. **教育理念**：4 个支柱改编号横排列表（01–04），平面无卡片堆叠，副文案每项 ≤ 2 行。
4. **课程体系**：重构 explorer——
   - 移动端：手风琴（accordion），4 个方向默认收起，点开看考试/服务详情；
   - 桌面：左侧 4 个方向竖排 tab + 右侧详情，浅色底（现为深色大面板，是色块拼接感来源之一）。
5. **校区与联系（新增，修复死链）**：现导航「校区联系」指向 `/#contact` 但首页没有该锚点。新增浅色板块：校区地址列表 + 电话/微信 + 二维码，同时承接转化。
6. **底部 CTA**：浅色一行式横条（文案 + 按钮左对齐），不再深蓝全屏背景图。

## 6. 名师列表页（移动端核心改造）

- 页头：去掉大面积深色 page-hero，改浅色紧凑页头（面包屑 + 标题 + 一行说明，约 160–200px 高）。
- 筛选条：学科胶囊横向可滑动，移动端吸顶（sticky）。
- **网格：手机固定 2 列**（卡片间距 12px），平板 3 列，桌面 4 列。
  老师照片比例统一（591×827），双列等宽等高，视觉上即整齐的双列流；
  若后续照片比例不一，再升级为真正的不等高瀑布流（CSS columns）。
- 教师卡重排：照片（无角标堆叠）→ 姓名+评分（同行，评分小字金色）→ 学历（≤2 行截断）→ 课程标签（≤2 个，其余省略）。
  删除学科角标（筛选条已表达学科），删除 hover 浮起金边框。

## 7. 教师详情页

- 移动端：照片宽度收窄居中（约 60% 宽），姓名/学历/评分先行，评分条从三色渐变改平面。
- 桌面保持左右栏，但整体换浅色体系。

## 8. 移动端转化

- 移动端屏幕底部增加常驻吸底条：「电话咨询 | 微信预约」两个按钮，滚动时出现，预约弹窗打开时隐藏。
- 预约弹窗移动端全屏化（bottom sheet 样式），二维码优先。

## 9. 教师评分数据维护文档

- 新建 `docs/教师评分数据.md`：一张 Markdown 表格，列为 姓名 / 学科 / 学习提升 / 责任心 / 个人魅力 / 综合分（自动）/ 数据状态（真实|占位）。
- `src/data/teachers.ts` 仍是运行时数据源；表格改完后由开发同步进代码（表格与代码字段一一对应，机械同步）。
- 当前所有评分标记为"占位"，文档内注明替换流程。

## 10. 清理清单（随重构一并处理）

- 删除 Google Fonts 与 tabler webfont 两个外部阻塞资源；
- 删除死代码：`.hero-stats` 样式、`logo: "/logo.png"` 死引用、未使用的 Unsplash 占位图数据；
- `bg.webp` 全站只出现一次（Hero），其余板块改纯色；
- 轮播组件 `FacultyCarousel` 整体移除（含每帧 scrollLeft 的耗电逻辑）。

## 11. 不改动的部分

- Next.js 架构、路由（`/`、`/faculty`、`/faculty/[name]`）、SSG 构建方式不变；
- 文案内容主体不变（只做精简压缩，不重写）；
- 上一轮的性能优化（WebP、懒加载、缓存头）全部保留；
- 不新增录取成果板块（按你的要求）。
